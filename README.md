## Laravel microservice highload-testing application

Это несложное приложение с web-интерфейсом на основе Laravel 12 для нагрузочного тестирования локальных микросервисов 
или для проверки работы балансировщиков нагрузки.

Оно выполняет выбранное количество асинхронных запросов на указанный URL. 
Количество асинхронных запросов ограничено только ресурсами RAM и CPU системы. 
В этой версии приложения есть возможность выполнять только запросы POST, но предусмотрена возможность 
расширения функционала добавлением любых других типов запросов, например gRPC или GraphQL.

### Requirements

Для работы приложения необходимо:

* PHP 8.5
* redis php-extension
* pdo php-extension
* pdo_pgsql php-extension
* Composer
* Docker
* Node v25.9
* Npm v11.12

### Installation

Для начала работы приложения нужно выполнить установку библиотек composer:

    composer install

Запустить сборку npm

    npm run build

Установить сервисы через Docker Compose:

    docker compose up

Выполнить миграции:

    php artisan migrate

### Launch

Для запуска приложения надо выполнить в двух разных терминалах консольные команды:

    php artisan serve

и

    php artisan queue:work

И переходить по адресу запущенного сервера в браузере.

Чтобы начать работать с приложением надо сначала зарегистрироваться, email не проверяется, можно использовать любой.
Зарегистрировавшись мы переходим на главную страницу "Dashboard", на ней можно выбрать настройки для нагрузочного теста
и запустить его.

### Queue

Запуск множества асинхронных параллельных процессов это дорогостоящая процедура требующая значительных 
ресурсов системы, как RAM так и CPU. Например запуск 100 процессов может занять около 32 секунд. 
И поэтому, чтобы не приходилось ждать ответа от сервера, в приложении используется Queue.

Множество механизмов приложения может быть доработано и оптимизировано для снижения стоимости запуска
параллельных процессов, это является направлением для развития этого приложения.

### Highloads

Выполненные нагрузочные тесты отображаются на странице "Highloads", здесь можно выбрать любой из них, чтобы просмотреть 
результаты тестирования.

Здесь также можно сгенерировать файлы с результатами тестирования в различных форматах,
CSV или HTML, после генерации эти файлы добавятся в директории:

    storage/reports/csv

или

    storage/reports/html

### Console reports

В приложении также есть возможность генерации файлов с результатами тестирования через консольную команду.
В этой версии приложения есть два формата таких файлов, CSV и HTML, но предусмотрена возможность добавления любых
других форматов.

Для генерации CSV файлов надо выполнить команду:

    php artisan reports:generate

или

    php artisan reports:generate --format="csv"

CSV файлы добавляются в директорию:

    storage/reports/csv

Для генерации HTML файлов надо выполнить команду:

    php artisan reports:generate --format="html"

HTML файлы добавляются в директорию:

    storage/reports/html

### Database

В проекте есть docker-compose файл с настроенной базой данных Postgres, но при необходимости можно 
настроить и любую другую бд.

### Logs

В логи добавляется различная информация о работе приложения:

* laravel.log - Общая информация о работе приложения и о выполненных запросах
* tests.log - В этот лог информация направляется при запуске тестов
